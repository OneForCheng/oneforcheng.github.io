如何提升工作效率？

2019-01-23
#工作 #效率
方法论

一、反思回顾——自己平时的学习或工作的效率高吗？

你是否经常遇到如下情况呢？

二、如何提升效率？

这里推荐一种方法：任务列表法+PDCA

由于已经有一篇文章——编程的精进之法——很好的介绍了这种方法，这里就不做过多描述。虽然这篇文章主要是将这个方法应用在编程之中，但其实是可以应用生活的方方面面。

三、与番茄工作法的区别

番茄工作法

描述：

  • 一个以集中注意力来提升工作效率的方法

如何做：

  • 30分钟内集中精力只做一件事情
  • 30分钟后，可以适当休息5分钟
  • 若中途中断，则此次番茄坏掉，应重新开始

原理：

  • 通过外在计时暗示自己集中精力做事

特点：

  • 集中注意力

注意要点：

  • （尽全力保证）不被中断

任务列表（PDCA）

描述：

  • 一个可以持续改进自己工作效率的闭环方法

如何做：

  • 罗列任务列表，将大的任务拆分为多个小任务（保证每个任务描述必须是明确的、相互独立的、可践行的）
  • 对每一项任务进行时间预估（最初预估不准不要紧，需要不断的经验积累不断改进）
  • 根据优先级进行排序、调整次序
  • 开始执行任务列表，针对每条任务都应该准确计时（最重要的是避免任务中断）
  • 记录完成所用时间
  • 完成后若与预期存在很大差异，则反思总结，（最重要的）并且产生具体的行动

原理：

  • 通过明确、有序的执行任务来避免混沌不清地做事
  • 通过外在计时暗示自己集中精力做事
  • 通过自我反馈、反思来不断持续改进不足之处

特点：

  • 明确的
  • 有序的
  • 集中注意力
  • 自我反馈改进

注意要点：

  • 任务列表（尽可能）穷尽（小步快走思想）
  • 任务列表完全独立
  • 不被中断

四、具体如何做呢？

对TaskList进行PDCA的完整样例模板：TaskList（Plan-预估时间|Do-实际花费时间|Check-原因或反思|Action-具体的行动）

Demo展示：

  • 阅读公众号（20|00:17:32）
    • 1篇细读
  • 制定本周计划、目标（15|00:16:23|有些计划描述不够明确，太过笼统|将笼统的计划拆分为更小的步骤描述）
  • 练习钢笔字
    • 临帖（10|00:10:00|注意力不够集中，临字的数量有点少）
    • 背临（20|00:10:00）
  • 练习键盘打字
    • 金山打字（15|00:15:00）
    • Typing Practice（15|00:15:00|准确率不高，应该优先保证准确率|若错误次数超过3次，重来）
  • 编程任务
    • 商品购物练习
      • 将输入的字符串数组转化为Orders数组（10|00:14:12）
      • 将订单Orders数组按商品编号进行合并，得到mergedOrders（10|00:23:03）
      • 根据合并后订单mergedOrders来获取清单详细
        • 判断商品是否属于“二赠一”的商品（10|00:07:34）
        • 获取某个订单mergedOrder的赠送数量（5|00:03:45）
        • 获取所购买商品的详细信息（15|00:32:32|超过预期时间，对JS的一些常用函数不熟练，查询资料所花时间比较多|对JS的一些常用方法需要花时间了解学习）
        • 获取所购买商品中属于优惠商品的详细信息（15|00:18:19）
        • 获取所购买商品优惠后的总价（5|00:05:23）
        • 获取所购买商品的优惠总价（5|00:06:19）
